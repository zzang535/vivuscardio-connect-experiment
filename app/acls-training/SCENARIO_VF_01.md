# VF-01 시나리오 연출 기획

## 시나리오 정의

**이름:** 일반 병동 VF 심정지 — 발견부터 첫 제세동 직후 CPR 재개까지

**한 문장 경험:** 야간 병동에서 환자 모니터 알람을 들은 간호사가 최초 반응자로 병실에 들어가 심정지를 확인하고, 코드팀 도착 후 VF를 판단하여 첫 제세동과 즉시 CPR 재개를 지시한다.

이 시나리오는 전체 ACLS 알고리즘을 가르치지 않는다. 약물, 반복 제세동, H's and T's, ROSC 후 처치는 후속 시나리오로 분리한다.

## 학습 목표

학습자는 제한된 정보와 시간 안에 다음 행동을 수행한다.

1. 환자 반응, 호흡과 맥박을 신속하게 확인한다.
2. 심정지를 인지하고 코드 블루 호출과 CPR 시작을 지시한다.
3. 코드팀 도착 후 압박을 유지하면서 모니터/제세동기 연결을 지시한다.
4. 압박 중단을 최소화하여 VF를 판독한다.
5. 제세동기 충전, 안전 확인과 shock을 지시한다.
6. shock 직후 맥박을 확인하지 않고 CPR을 즉시 재개한다.

## 대상과 역할

- 대상: 신규 간호사, 일반 병동 간호사, 응급실 간호사
- 학습자 역할: 병동 간호사이자 코드팀 도착 전 현장 리더
- 환자: 68세 남성, 심혈관 질환으로 입원 중인 성인 환자
- 시간대: 야간, 병실에는 환자와 학습자만 있는 상태로 시작
- 팀 도착 후 등장 인물: 압박 담당 간호사, 제세동 담당 간호사, 기도 담당자

## 범위

### 시작점

병실 밖 스테이션에서 모니터 알람이 발생하고 학습자가 병실로 진입한다.

### 종료점

첫 shock 후 5초 안에 CPR 재개가 확인되면 장면을 종료하고 디브리핑으로 전환한다.

### 포함

- 환자 접근
- 반응 확인
- 호흡 및 맥박 확인
- 코드 블루 호출
- CPR 시작
- 코드팀 및 제세동기 도착
- 패드/모니터 연결
- VF 판독
- 충전과 안전 확인
- 첫 shock
- 즉시 CPR 재개

### 제외

- IV/IO 확보 및 약물
- 두 번째 이후 CPR 주기와 제세동
- epinephrine 및 항부정맥제
- 가역적 원인 감별
- ROSC와 소생 후 처치
- 실제 소생 중단 판단

## 경험 원칙

1. **버튼보다 상황이 먼저다.** 사용자는 처음부터 전체 처치 목록을 보지 않는다.
2. **시간은 결과를 만든다.** 지연하면 팀이 재보고하고 환자 상태와 채점에 반영된다.
3. **소리는 정보다.** 알람, 대사, 충전음과 메트로놈이 다음 판단의 단서가 된다.
4. **행동은 장면 안에서 일어난다.** 명령 후 의료진이 이동하고 준비하며 완료를 보고한다.
5. **오답도 진행된다.** 치명적인 조작 오류를 제외하면 즉시 정답을 알려주지 않고 결과를 기록한다.
6. **한 장면을 밀도 있게 만든다.** 전체 ACLS 메뉴보다 첫 60초의 긴장감과 판단을 우선한다.

- 점수와 알고리즘 진행률은 훈련 중 표시하지 않는다.
- 화면 상단의 경과시간은 작게 제공하며 카운트다운 정답 힌트로 사용하지 않는다.
- 팀 보고 자막은 음성과 동시에 나타났다가 최근 보고 영역에 누적된다.
- 클릭 가능한 모든 객체를 상시 강조하지 않는다. 시선 유도가 필요할 때만 알람, 움직임과 조명을 사용한다.

## 장면 연출 타임라인

시간은 권장 연출 기준이며 사용자의 행동에 따라 일부 장면이 앞당겨지거나 지연된다.

### Scene 1 — 알람과 진입 (00:00–00:06)

**화면**

- 어두운 야간 간호 스테이션의 환자 모니터 일부를 클로즈업한다.
- 특정 병상의 붉은 알람 표시가 점멸한다.
- 1초 후 병실 전경으로 짧게 전환하고 환자가 움직이지 않는 모습을 보여준다.

**소리**

- 환자 모니터 high-priority alarm 반복
- 병동의 낮은 환풍기 소리와 멀리서 들리는 카트 소리
- 배경음악은 사용하지 않는다.

**사용자 행동**

- 알람 병상을 선택하여 병실로 진입한다.
- 5초 동안 선택하지 않으면 담당 간호사 NPC가 “해당 병상 확인이 필요합니다”라고 알린다.

**평가**

- 알람 인지부터 병실 진입까지 걸린 시간을 기록한다.

### Scene 2 — 환자 확인 (00:06–00:16)

**화면**

- 병상 옆 고정 사선 구도. 환자는 눈을 감고 움직이지 않는다.
- 환자를 선택하면 `반응 확인`, `호흡·맥박 확인`, `도움 요청`이 문맥 행동으로 열린다.
- 반응 확인 시 간호사가 어깨를 두드리는 짧은 동작을 수행한다.
- 호흡 확인 시 상체와 경동맥 부위로 화면이 짧게 확대된다.

**소리와 대사**

- 학습자 명령 후 NPC 음성: “환자분, 괜찮으세요?”
- 결과 보고: “반응 없습니다. 정상 호흡 없고 맥박 촉지되지 않습니다.”
- 확인 중 모니터 알람은 계속되지만 음량을 낮춰 대사가 들리게 한다.

**시간 규칙**

- 호흡과 맥박 확인은 최대 10초 동안 진행된다.
- 10초가 넘어가면 보고 없이 자동 완료하지 않는다. “맥박이 확실하지 않습니다”라는 재촉 보고가 발생하고 지연을 기록한다.

**평가**

- 반응 확인 여부
- 호흡과 맥박 확인 여부
- 전체 확인 소요시간

### Scene 3 — 심정지 선언과 초기 대응 (00:16–00:26)

**화면**

- 확인 결과가 나온 직후 환자 주변에 `코드 블루 호출`, `CPR 시작`, `산소 준비` 행동이 열린다.
- 코드 호출 시 병실 상단 표시등이 켜지고 복도 방향에서 팀이 접근하는 모습이 보인다.
- CPR 지시 시 간호사가 침상 난간을 내리고 환자 가슴 중앙에서 압박을 시작한다.
- 실제 압박 주기를 느낄 수 있도록 상체와 팔이 반복 움직인다.

**소리와 대사**

- “코드 블루 호출합니다. 제세동기 가져오세요.”
- 병동 코드 블루 방송과 짧은 호출 벨
- CPR 시작과 동시에 110 bpm 메트로놈
- 압박 담당자의 30회 단위 숫자 세기는 사용하지 않는다.

**동시성**

- 코드 호출과 CPR 시작은 어느 순서든 지시할 수 있다.
- 코드 호출 후 팀 도착까지 7초가 걸린다.
- CPR 시작이 지연되어도 팀 도착 타이머는 독립적으로 진행된다.

**평가**

- 심정지 확인부터 코드 호출까지 지연
- 심정지 확인부터 첫 압박까지 지연
- 초기 CPR 미실시 시간

### Scene 4 — 팀 도착과 장비 연결 (00:26–00:38)

**화면**

- 병실 문이 열리고 제세동기 카트를 끄는 팀원 2명과 기도 담당자 1명이 진입한다.
- 사용자가 팀원 또는 장비를 선택하면 문맥 명령이 표시된다.
- `압박 교대`, `제세동 패드 부착`, `산소·앰부 환기`를 동시에 지시할 수 있다.
- 제세동 담당자가 환자의 의복을 열고 패드를 부착하며 케이블이 장비까지 이어진다.
- CPR은 패드 부착 중에도 계속된다.

**소리와 대사**

- “코드팀 도착했습니다.”
- “압박 이어받겠습니다.”
- “패드 부착 중입니다.”
- 완료 시 “모니터 연결됐습니다. 리듬 확인 가능합니다.”
- 카트 바퀴, 패드 포장 개봉과 케이블 연결음

**평가**

- CPR 유지 여부
- 패드 부착 지시 시간
- 동시 작업 활용 여부
- 불필요하게 CPR을 중단했는지 여부

### Scene 5 — 리듬 확인과 충전 (00:38–00:50)

**화면**

- `리듬 확인` 명령 시 압박 담당자가 손을 떼고 10초짜리 중단시간 측정이 시작된다.
- 모니터가 확대되어 VF 파형을 명확히 보여준다.
- 사용자는 `Shockable`, `Non-shockable` 중 리듬 분류를 선택한다.
- Shockable 선택 후 `200 J 충전`을 지시할 수 있다.
- 충전 중에는 CPR을 재개할 수 있으며, 팀원이 “충전 중입니다. 압박 계속하세요”라고 보고한다.

**소리와 대사**

- 압박 중단과 함께 메트로놈 정지
- “리듬 확인합니다.”
- 충전 지시 후 상승하는 제세동기 충전음
- 충전 완료음과 “200 J 충전 완료” 보고

**시간 규칙**

- 리듬 확인은 최대 10초 이내를 목표로 한다.
- 충전 시간은 3초이며 CPR과 병행 가능하다.
- 압박이 중단된 모든 시간을 누적한다.

**평가**

- VF 판독 정확성
- 리듬 확인 소요시간
- 충전 중 CPR 재개 여부
- 누적 CPR 중단시간

### Scene 6 — 안전 확인, shock, CPR 재개 (00:50–01:00)

**화면**

- 충전 완료 후 학습자는 `모두 물러남 확인`을 먼저 지시해야 한다.
- 의료진이 환자와 침상에서 손을 떼는 장면을 각각 확인할 수 있다.
- 안전 확인 완료 후 shock 버튼이 활성화된다.
- shock 순간 화면과 환자에 짧은 시각 효과가 발생하지만 과장된 신체 움직임은 사용하지 않는다.
- shock 직후 모니터를 확인하거나 맥박을 확인하는 선택도 가능하지만 부적절한 행동으로 기록한다.
- 올바른 `CPR 즉시 재개` 지시 시 압박이 다시 시작된다.

**소리와 대사**

- 팀 리더: “모두 물러나세요. 산소 떨어졌습니다.”
- 각 팀원: “물러났습니다.”
- 제세동기 shock 효과음
- shock 후 2초 동안 별도 음향 없이 상황 판단 공간을 준다.
- CPR 재개 시 메트로놈과 “압박 재개했습니다” 보고

**종료 규칙**

- shock 후 5초 안에 CPR을 재개하면 정상 완료
- 5초를 넘기면 재촉 보고 후 자동 종료하지 않고 최대 10초까지 기다린다.
- 10초까지 CPR을 재개하지 않으면 NPC가 압박을 재개하고 `중대한 지연` 결과로 종료한다.

**평가**

- shock 전 안전 확인 완료 여부
- shock 수행 정확성
- shock부터 CPR 재개까지의 시간
- shock 후 불필요한 맥박/리듬 확인 여부

## 상황 진행 엔진

시나리오는 단순한 경과시간이 아니라 장면과 사건으로 진행한다.

```ts
type SceneId =
  | "alarm"
  | "patient-check"
  | "arrest-response"
  | "team-arrival"
  | "rhythm-check"
  | "defibrillation"
  | "debrief";

type ScheduledCue = {
  atMs: number;
  scene: SceneId;
  type: "visual" | "voice" | "sound" | "team-action" | "deadline";
  payload: string;
  cancelledBy?: string[];
};
```

- 각 장면은 진입 조건, 사용자 행동, 자동 사건, 제한시간과 종료 조건을 가진다.
- 사용자가 먼저 행동하면 예정된 재촉 음성은 취소한다.
- 제한시간을 넘겨도 타이머만 흐르게 두지 않고 재보고, NPC 개입 또는 장면 종료가 발생한다.
- 시나리오 최대 길이는 90초로 제한한다.
- 일시정지는 훈련 시간과 모든 예약 사건, 애니메이션 및 사운드를 함께 멈춘다.

## 사용자 입력 모델

전체 ACLS 행동을 한꺼번에 노출하지 않는다.

| 선택 대상 | 표시 가능한 문맥 행동 예시 |
| --- | --- |
| 환자 | 반응 확인, 호흡·맥박 확인, CPR 지시 |
| 병실 호출 장치 | 도움 요청, 코드 블루 호출 |
| 압박 담당자 | CPR 시작, 중단, 교대 |
| 제세동 담당자 | 패드 부착, 리듬 확인, 충전, shock |
| 기도 담당자 | 산소 준비, 앰부 환기, 환자에게서 분리 |
| 모니터 | 리듬 확대, Shockable/Non-shockable 분류 |

키보드 단축키나 하단 명령 팔레트는 접근성 보조 수단으로 제공할 수 있지만, 장면 안의 대상 선택과 동일한 문맥 행동만 표시한다.

## 음향 명세

### 반드시 필요한 음원

| 분류 | 음원 | 반복/종료 조건 |
| --- | --- | --- |
| 환경 | 야간 병동 room tone | 시나리오 전체, 낮은 음량 |
| 알람 | 고우선순위 환자 모니터 알람 | 초기부터 모니터 연결/상황 전환까지 |
| 방송 | 코드 블루 호출 방송 | 코드 호출 직후 1회 |
| CPR | 110 bpm 메트로놈 | CPR 진행 중에만 반복 |
| 장비 | 카트, 패드 포장, 케이블 연결 | 해당 애니메이션과 동기화 |
| 제세동 | 충전 상승음, 완료음, shock음 | 각 상태 전환 시 1회 |
| 음성 | 팀원 보고 및 재촉 | 예약 cue 또는 행동 완료 시 |

### 오디오 원칙

- 첫 사용자 입력 또는 시작 버튼 이후 오디오 컨텍스트를 활성화한다.
- 동시에 재생되는 음성은 하나만 허용하고 중요한 보고를 우선한다.
- 음성 재생 중 알람과 환경음은 자동으로 음량을 낮춘다.
- 음소거, 전체 음량과 자막을 제공한다.
- 모든 음성 정보는 동일한 시점에 자막과 이벤트 로그로 제공한다.

## 시각 자산 제작 목록

### 배경

- 야간 병동 스테이션
- 동일 화풍과 시점의 병실 기본 배경
- 열린 문과 닫힌 문 상태
- 코드 블루 표시등 on/off

### 환자

- 무반응 기본 자세
- 반응 확인 중
- 호흡·맥박 확인 중
- CPR 중 상체 상태
- 패드 부착 상태
- shock 상태

### 의료진

- 최초 반응 간호사: 대기, 접근, 반응 확인, 맥박 확인, CPR
- 압박 담당자: 진입, 준비, 압박, 손 뗌
- 제세동 담당자: 카트 이동, 패드 준비, 부착, 충전, 안전 확인
- 기도 담당자: 진입, 산소 준비, 환기, 산소 분리

### 장비 및 효과

- 모니터 알람/VF 확대 화면
- 제세동기 이동·연결·충전·완료 상태
- 패드 및 연결 케이블
- 앰부백과 산소 라인
- 대상 선택, 작업 중, 완료 표시
- shock 플래시와 CPR 리듬 표시

모든 자산은 동일한 카메라 구도, 조명과 화풍을 공유해야 한다. 사실적인 배경 위에 추상적인 CSS 인물을 겹치는 방식은 사용하지 않는다.

## 디브리핑

시나리오 종료 후 전체 점수보다 행동의 시간 관계를 먼저 보여준다.

```text
00:05  병실 진입
00:11  심정지 확인                 권장 범위
00:13  코드 블루 호출              +2초
00:15  CPR 시작                    +4초
00:28  패드 부착 완료              CPR 유지
00:39  리듬 확인                   중단 7초
00:47  VF 판독 / 충전              정확
00:53  첫 shock                    안전 확인 완료
00:57  CPR 재개                    shock 후 4초
```

평가 항목:

- 심정지 인지 정확성
- 코드 호출 지연
- 첫 압박 지연
- VF 판독 정확성
- 누적 CPR 중단시간
- shock 전 안전 확인
- shock 후 CPR 재개시간
- 불필요한 행동

결과는 `잘한 점`, `환자 결과에 영향을 준 지연`, `다시 시도할 한 가지 목표`로 요약한다.

## 프로토타입 완료 기준

- 전체 재생시간이 60~90초 안에 종료된다.
- 아무 행동을 하지 않아도 재촉, NPC 개입 또는 제한 종료가 발생한다.
- 첫 장면부터 shock 후 CPR 재개까지 장면 전환이 연결된다.
- 모니터 알람, 코드 방송, CPR 메트로놈, 충전 및 shock음이 상태와 동기화된다.
- 팀원의 핵심 보고가 음성과 자막으로 동시에 제공된다.
- 의료진의 위치와 자세가 현재 처치를 명확히 보여준다.
- CPR과 장비 준비가 동시에 진행된다.
- 훈련 중 점수와 정답 힌트가 노출되지 않는다.
- 모든 행동과 자동 사건이 하나의 시간축 로그에 기록된다.
- 종료 후 실제 수행 타임라인을 확인하고 동일 시나리오를 다시 시작할 수 있다.

## 구현 순서

1. Scene 1~2: 알람, 병실 진입, 환자 확인과 음성/자막
2. Scene 3: 코드 호출, CPR 애니메이션과 메트로놈
3. Scene 4: 팀 진입, 동시 작업과 패드 연결
4. Scene 5: VF 판독, 압박 중단시간과 충전
5. Scene 6: 안전 확인, shock, CPR 재개와 종료
6. 이벤트 로그 기반 디브리핑
